## Top diferenças de rota visual N2/N3
- N2 é muito próximo entre Web e Flutter, mas precisa prova de conexão real com a aula e T02.
- N3 diverge: Web usa serverFn/Gemini Flash Lite; Flutter usa decisão local e SVG sintético.
- A diferença N3 é arquitetura/plataforma, mas exige decisão se o juiz remoto é saúde obrigatória.

## Top diferenças de pipeline visual
- S12 é portado, mas logs e provas de runtime diferem.
- SVG grátis passa fora de gate de custo nos dois, mas precisa prova na aula real.
- Prefetch background bloqueia AI paga; precisa teste anti-cobrança em Flutter/API.

## Top diferenças de templates matemáticos
- Ambos têm cinco templates principais.
- Web tem formula-parser TS listado; Flutter não mostra equivalente direto.
- Diferença principal é prova de roteamento template -> aula -> scroll.

## Top diferenças de imagem paga
- API própria exige acceptedOfferId; Web gera idempotência por hash de lessonKey/prompt.
- Flutter usa offerId timestamp como acceptedOfferId/idempotencyKey.
- Falta prova APK real de aceite, cobrança, retry e erro.

## Top diferenças de crédito/cobrança
- API reserva/captura/reembolsa em controllers próprios.
- Web usa RPC Supabase e circuit breaker antes de cobrar.
- Conta infinita pode mascarar bug de cobrança real se não houver teste com conta normal.

## Top diferenças de idempotência
- API tem operationId e cache por user/key/prompt.
- Web idempotencyKey deriva de hash estável.
- Flutter usa offerId local; se recriar oferta, a idempotência muda.

## Top diferenças de endpoint/segurança
- API usa protect + resource owner + rate limit.
- Web usa verifyAuth local por rota.
- Precisam provas com token inválido, dono errado e replay.

## Top diferenças de áudio/TTS
- Web tem fallback real via SpeechSynthesis.
- Flutter Android toca dataUrl remoto, mas fallback TTS local retorna false.
- Botão Flutter de áudio visível é toggle de preferência em widget separado; controller de tocar/parar é outro órgão.

## Top riscos de áudio falso
- NoopAudioPlaybackAdapter existe e pode mascarar teste se vazar por injeção errada.
- PlatformAudioAdapter não fala localmente se áudio remoto falhar.
- Estado 'falando' depende de callbacks; erro assíncrono pode parecer concluído sem áudio audível.

## Top riscos de regressão
- Duplo toque em imagem paga.
- Timeout de imagem e política de refund diferente.
- Cache/idempotência por offerId instável.
- Áudio remoto falhar e fallback mobile não tocar.
- Teste com crédito infinito esconder falta de saldo normal.
