## Top diferenças de cliente HTTP
- Flutter usa `DartIoSimHttpTransport`; Web usa `fetch`/serverFn.
- Flutter não mostra `x-request-id` nos headers lidos.
- Multipart Flutter é manual; Web usa FormData/serverFn.

## Top diferenças de timeout/erro
- Imagem: Flutter 60s, API 120s, Web tem polling/circuit breaker.
- Áudio: Flutter 45s, API 90s, Web 45s com AbortController.
- Erros HTTP do Flutter sobem como `SimExternalAiException`; Web às vezes cai para fallback.

## Top diferenças de auth/resource owner
- API própria usa requireAuth + resourceOwners.
- Web usa Supabase getClaims por rota.
- Resource owner usa chaves derivadas de lesson/cache/media; precisa prova contra usuário cruzado.

## Top diferenças de cacheKey/hash
- Imagem API inclui userId+lessonHash+promptHash.
- Áudio API inclui lessonKey+lang+hash(text).
- Cache local Flutter usa hashCode Dart para áudio, que não é contrato servidor.

## Top diferenças de voz/idioma
- Voz por idioma bate em pt/en/es/fr, mas Web usa fallback browser.
- Flutter converte stableLang em BCP47 no core.
- Servidor aceita stable_lang/language/lang com fallback pt.

## Top diferenças de preferência de áudio
- Flutter tem SharedPrefs storage, mas default do AudioPreference é memória.
- Web usa localStorage e evento cross-tab.
- Áudio desligado no Flutter evita chamada; Web também cancela quando a preferência desliga.

## Top diferenças de parar áudio
- Flutter para áudio em selecionar, sinal e avançar via controller.
- Web para no unmount pelo hook.
- Dispose da tela principal Flutter não mostrou chamada direta de stop na prova lida.

## Top diferenças de permissões/dependências
- AndroidManifest inclui internet/camera/read media images e cleartext.
- Dependências existem no pubspec: audioplayers, image_picker, file_picker, flutter_svg, image.
- Web usa permissões do navegador via input file/capture.

## Top diferenças de foto da dúvida/anexos
- Dúvida Flutter valida payload data:image, mas picker/conversão real fica na tela.
- Web converte FileReader + compressão no sheet.
- API de anexos atual é bem mais simples que o Web OCR/vision.

## Top diferenças de avatar/bolha/onda
- Web FixedBubble é aria-hidden e pointer-events-none.
- Flutter FixedBubble separado tem GestureDetector; a implementação inline da aula usa IgnorePointer.
- Avatar/boca/onda dependem de speaking/falando e precisam prova de áudio real.

## Top riscos de regressão
- Header sem requestId dificulta diagnóstico de mídia.
- Timeout divergente pode gerar estados diferentes entre app e API.
- Anexo de imagem pode virar metadata na API, diferente do Web.
- Áudio pode não parar no dispose/lifecycle Android.
- Cache/hash divergente pode reutilizar áudio em idioma errado.
