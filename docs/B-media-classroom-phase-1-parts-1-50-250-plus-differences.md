## Top diferenças de visual_trigger

- Entrada legada textual no Web versus JSON estruturado no Flutter/API.
- N3 remoto no Web versus N3 local generico no Flutter.
- `aspect_ratio` legado do Web nao aparece no modelo Flutter rico.
- Servidor valida `visual_trigger` basico, mas nao valida profundamente `math_template`.
- Flutter tem dois parsers/caminhos (`LessonVisualTrigger.fromJson` e `validatedLessonContentFromJson`) com rigor diferente.

## Top diferenças de SVG

- Web sanitiza cortando trechos perigosos; Flutter rejeita SVG suspeito inteiro.
- Web N3 limita 12 KB; Flutter S12 permite 100 KB.
- Web exige viewBox no prompt N3; Flutter sanitizador nao exige viewBox.
- Flutter renderiza com `SvgPicture.string`; Web usa `<img src=dataUrl>`.
- SVG Flutter nao dispara scroll/onLoad especifico.

## Top diferenças de imagem gerada

- API propria usa Gemini inline; Web legado usa Replicate/Lovable gateway.
- API propria nao tem circuit breaker nem retry provider igual Web.
- Flutter so consome `dataUrl`; servidor retorna metadados de cobranca/cache.
- Flutter comprime client-side; servidor nao entrega variante menor.
- Timeout cliente/servidor esta desalinhado.

## Top diferenças de imagem no celular

- Altura fixa 168 no Flutter contra max-height 360 no Web.
- Oferta Flutter com Row fixa contra flex-wrap do Web.
- Painel Flutter adiciona texto/status e reduz area util.
- Conteudo Flutter nao evidencia max-width tablet.
- Zoom/fonte alto pode quebrar oferta por altura fixa.

## Top diferenças de imagem e scroll

- Web usa ResizeObserver.
- Web usa onLoad da imagem.
- Flutter so faz scroll por fase/snapshot.
- Imagem atrasada pode deslocar desafio.
- Feedback/botao avancar podem sair do viewport apos carga de imagem.

## Top riscos de regressão

- Imagem anterior aparecer em item/layer novo.
- Oferta recusada em uma aula afetar outra.
- SVG sem viewBox quebrar proporcao.
- DataUrl enorme travar memoria.
- Decode quebrado aparecer como imagem pronta.
- Cliente desistir antes do servidor concluir.
- Falta de evento `IMAGE_READY/FAILED` no StudentLearningState.
- Historico perder dataUrl por usar `Image.network`.
- Painel de loading de imagem ser confundido com erro/aula travada.
- App fechar antes de persistencia async do cache.
