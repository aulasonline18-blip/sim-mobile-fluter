## Top diferenças de áudio lifecycle
- Saída de tela: Web tem cleanup explícito no hook; Flutter tem stop no controller, mas a prova lida não mostrou cleanup equivalente na tela principal.
- Background Android: não há política explícita provada nesta auditoria.
- Erro remoto: Web cai para SpeechSynthesis; Flutter depende do adapter, e o adapter real retorna false para TTS local.

## Top diferenças de bolha/avatar
- Web renderiza FixedBubble por audioEnabled/falando; Flutter usa session.audioPlaying e possui também widget separado não necessariamente conectado.
- Avatar Web anima waveform por speaking; Flutter tem widget separado e a auditoria precisa provar conexão com estado real.
- Bolha Flutter pode competir com botão de fonte/área inferior por ser overlay no Stack.

## Top diferenças de cache local
- Flutter persiste texto/audioText e remove imagem do cache persistido.
- Web documenta IndexedDB/cache com imagem|null e assinatura de atualização.
- Ambos limitam cache de aula, mas a prova de reabertura mobile com imagem precisa APK.

## Top diferenças de cache servidor
- API usa cache em memória com limite, sem TTL explícito.
- Imagem e áudio compartilham o media-cache criado no router.
- Restart do servidor perde cache de mídia.

## Top diferenças de StudentLearningState
- Flutter tem StudentAudioState typed serializado em audio_typed.
- Eventos de mídia entram no events log, mas requestId não aparece no payload de mídia local.
- API persiste state completo por usuário e highWaterMark, mas não entende semanticamente mídia.

## Top diferenças de replay/sync
- Flutter reconstrói events e audio_typed no fromJson.
- Server persiste/tomba state inteiro; cache de mídia fica separado.
- Deletar/renomear aula não prova limpeza/reatribuição de cache local de mídia.

## Top diferenças de LessonOrchestrator
- Texto resolve primeiro; imagem roda em background.
- Flutter usa ImageSequentialQueue simples; Web tem textInflight + imageInflight e promoção de prioridade documentada.
- onAudioTextReady prepara áudio sem iniciar playback.

## Top diferenças de ReadyWindow/prefetch
- Slots A/B/C têm prioridade A active, B/C background no Flutter.
- Web documenta que imagem/áudio não bloqueiam próxima aula.
- Falta prova APK de que mídia de B/C não aparece no slot A.

## Top riscos de áudio falso
- Bolha pode seguir estado, não áudio audível real.
- Noop/testes podem mascarar playback real.
- Falha remota no Android pode terminar sem fala local.

## Top riscos de mídia no item errado
- Cache por chave errada pode associar mídia a item/layer diferente.
- Renomear/deletar aula não limpa cache de mídia nesta prova.
- B/C prefetch precisa prova de não promover mídia errada para A.

## Top riscos de regressão
- Remover cleanup de áudio ao navegar.
- Persistir imagem grande no cache local sem política.
- Dedupe de imagem sem imageInflight no Flutter.
- Server cache em memória mascarar perda após restart.
- ReadyWindow preparar mídia paga indevida.
